In the late 2000s when building Places and the Awesomebar in to Firefox, we talked a lot about the power of revisitation, and imagined these features would lead in powerful directions. We had everything from task/navigation/trails to full-text-indexing to extensibility APIs all in the original plan. Developing the software turned out to be so slow and error prone that almost all of that was dropped. Nearly 20 years after that work started, Firefox doesn't have any of those things. Remarkably, it still has tags in the bookmarks UI... but not in the bookmarks API for extensions. We knew that the local profile was gold mine, mostly unsurfaced to users. There were a few attempts to rekindle that particular fire, but as of today neither Firefox nor any other major browser has treated this store of user data in that way - let alone *connecting* users with these features.
