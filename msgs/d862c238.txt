Found the root cause. When a web page is open in the webview (canvas mode), keyboard events go to the webview guest's `before-input-event`. That handler only handles navigation shortcuts (Cmd/Ctrl+L, R, [, ]) — it never calls `handleLocalShortcut`. So registered local shortcuts like Ctrl+K simply don't fire when a web page has focus.

On macOS this is masked because the global shortcut (Option+Space) works. On Linux the global shortcut fails (Wayland doesn't support it, or Alt+Space is grabbed by the WM), so there's no fallback.
